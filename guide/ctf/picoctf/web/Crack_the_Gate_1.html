Crack the Gate 1

Name: Crack the Gate 1
Description: We're in the middle of an investigation. One of our persons of interest, ctf player, is believed to be hiding sensitive data inside a restricted web portal. We've uncovered the email address he uses to log in: ctf-player@picoctf.org. Unfortunately, we don't know the password, and the usual guessing techniques haven't worked. But something feels off... it's almost like the developer left a secret way in. Can you figure it out? The website is running here. Can you try to log in?
Author: Yahaya Meddy
Tags: Easy, Web Exploitation, picoMini by CMU-Africa, browser_webshell_solvable
Challenge from: picoMini by CMU-Africa
Hints:
1. Developers sometimes leave notes in the code; but not always in plain text.
2. A common trick is to rotate each letter by 13 positions in the alphabet.

Theory

According to the description, to get the flag we have to enter into an account with an e-mail address and without the password, so we don't know anything from here yet, so let's enter to the site and view the source code with Ctrl+U of the webpage to know how the page works and maybe be able to get the password to log in.

Solution

So we'll open the website and check the source code of the webpage:

Oh look at that, it's a comment with probably important information, but it seems to be rotated as it remains with normal text structure:

Caesar Cipher (online tool i think)

INPUT: ABGR: Wnpx - grzcbenel olcnff: hfr urnqre "K-Qri-Npprff: lrf

SHIFT: FIND OUT

OUTPUT:
>13 (<13) - NOTE: Jack - temporary bypass: use header "X-Dev-Access: yes

Okay yup, we just need to do everything normally, just put the e-mail address we got, the password can be virtually anything, but then we need to use BurpSuite—a tool used to change request headers when doing internet stuff—and just add that `X-Dev-Access` header. And finally, press the forward button to make the modified request go through and get the flag without ever using a password:
