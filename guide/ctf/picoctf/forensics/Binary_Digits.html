Binary Digits

Name: Binary Digits
Description: This file doesn't look like much... just a bunch of 1s and 0s. But maybe it's not just random noise. Can you recover anything meaningful from this? Download the file here.
Author: Yahaya Meddy
Tags: Easy, Forensics, picoCTF 2026
Challenge from: picoCTF 2026
Files: digits.bin

Theory

According to the description, to get the flag we have to find it somewhere inside a file with a bunch of binary digits. So we'll download the file on solution, we don't know much about what might be on the file, but if it's big we'll need to decode the beginning only because it might be a lot.

Solution

Okay so it is kind of big, we'll use less to only view the top of the file and see what info we can get from the header:

shukularuni@SHUKULAP:~$ cat digits.bin | less
111111111101100011111111111000000000000000010000010010100100011001001001010001100000000000000001000000010000000000000...
...

Now let's just convert that binary to text (or well, hexdump to know what it is exactly) and see what it might be with CyberChef:

BINARY TO HEXDUMP (CyberChef)

INPUT: 111111111101100011111111111000000000000000010000010010100100011001001001010001100000000000000001000000010000000000000...

OUTPUT:
00000000  ff d8 ff e0 00 10 4a 46 49 46 00 01 01 00 00  |yOya..JFIF.....|
...

Oh this is familiar! It's all a JPEG image file, this is because it uses the JFIF format and also yOya (usually has accents and stuff) is there in the beginning as a clear sign, all we need to do now is convert the entire file from binary to text and change the file extension to jpg or jpeg. If you do that you'll get this image:

picoCTF{h1dd3n_1n_th3_b1n4ry_cc2099d3}

There we go! That's the flag.
