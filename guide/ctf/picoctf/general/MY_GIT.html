MY GIT

Name: MY GIT
Description: I have built my own Git server with my own rules! You can clone the challenge repo using the command below. git clone ssh://git@foggy-cliff.picoctf.net:61799/git/challenge.git Here's the password: 550851c0 Check the README to get your flag!
Author: Darkraicg492
Tags: Easy, General Skills, picoCTF 2026
Challenge from: picoCTF 2026
Hints:
1. How do you specify your Git username and email?

Theory

According to the description, to get the flag we need to do something with the git repository we are given. We can clone the repository into our machine with the command on the description and that password, then we'll check that README to know how to get the flag. So, no more information here in the description to go off of, so we'll head onto the solution part.

Solution

Enter the git command from the description to clone the repository and look at the README file:

shukularuni@SHUKULAP:~$ git clone ssh://git@foggy-cliff.picoctf.net:61799/git/challenge.git
Cloning into 'challenge'...
git@foggy-cliff.picoctf.net's password: 550851c0
remote: Enumerating objects: 3, done.
remote: Counting objects: 100% (3/3), done.
remote: Compressing objects: 100% (2/2), done.
remote: Total 3 (delta 0), reused 0 (delta 0)
Receiving objects: 100% (3/3), done.

shukularuni@SHUKULAP:~$ cd challenge/

shukularuni@SHUKULAP:~/challenge$ ls
README.md

shukularuni@SHUKULAP:~/challenge$ cat README.md
# MyGit

### If you want the flag, make sure to push the flag!

Only flag.txt pushed by ```root:root@picoctf``` will be updated with the flag.

GOOD LUCK!

Oh cool I get it. The wording makes it a little confusing, but basically we need to send a flag.txt file as if we are the root account, so we'll change the config to have our username as root and the email it says there, then we put something whatever on the flag.txt file, and finally we add it and push it to get the flag in some way:

shukularuni@SHUKULAP:~/challenge$ git config user.name "root"

shukularuni@SHUKULAP:~/challenge$ git config user.email "root@picoctf"

shukularuni@SHUKULAP:~/challenge$ echo "thingamabob">  flag.txt

shukularuni@SHUKULAP:~/challenge$ git add flag.txt

shukularuni@SHUKULAP:~/challenge$ git commit -m "hallo"
[master 47803cb] hallo
 1 file changed, 1 insertion(+)
 create mode 100644 flag.txt

So if we push that change of a file creation as a root user, and we get some funny message from remote (the flag!):

shukularuni@SHUKULAP:~/challenge$ git push
