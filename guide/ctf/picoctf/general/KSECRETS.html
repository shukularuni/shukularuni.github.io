KSECRETS

Name: KSECRETS
Description: We have a kubernetes cluster setup and flag is in the secrets. You think you can get it? Please wait for a minute for the application to be configured! Kubernetes is running at: green-hill.picoctf.net : 61181 You can find your configuration file here.
Author: Darkraicg492
Tags: Medium, General Skills, picoCTF 2026
Challenge from: picoCTF 2026
Files: kubeconfig.yaml
Hints:
1. Where are secrets usually stored in Kubernetes
2. How are Kubernetes secrets stored internally? Can you decode them?
3. Please ignore TLS

Theory

According to the description, to get the flag we have to look through a Kubernetes cluster in the server we are given on the description and extract a flag from there. We're given a file kubeconfig.yaml and told the flag is stored in Kubernetes secrets, so we'll skip this part to go check the file on the solution part.

Solution

When we open the kubeconfig file we see that the server is set to 127.0.0.1:6443, which is just a localhost, so it won't work initially. We can edit the file to change the localhost to the real server that we're given on the description, so we'll swap the localhost for the real server then export it to be able to use the updated file:

shukularuni@SHUKULAP:~$ sed 's|https://127.0.0.1:6443|https://green-hill.picoctf.net:61181|' kubeconfig.yaml>  kubeconfig_ctf.yaml

shukularuni@SHUKULAP:~$ export KUBECONFIG=kubeconfig_ctf.yaml

Now we'll list all of the secrets in the Kubernetes cluster, we add the all namespaces option to list all secrets so that we don't miss any, then the insecure skip option so that we skip TLS certificate validation:

shukularuni@SHUKULAP:~$ kubectl get secrets --all-namespaces --insecure-skip-tls-verify
NAMESPACE     NAME                       TYPE                               DATA   AGE
kube-system   chart-values-traefik       helmcharts.helm.cattle.io/values   1      6m45s
kube-system   chart-values-traefik-crd   helmcharts.helm.cattle.io/values   0      6m45s
kube-system   k3s-serving                kubernetes.io/tls                  2      6m49s
picoctf       ctf-secret                 Opaque                             1      6m39s

Well, that's funny. Coincidentally there's a secret literally by the name of ctf-secret. So it's pretty easy to assume that's the one we're looking for, so to view that secret specifically we'll use this command that gets the secret that has the name ctf-secret and in the namespace of picoctf, then again we'll skip TLS certificates and we'll point to the yaml file, and we should get the secret from the server:

shukularuni@SHUKULAP:~$ kubectl get secret ctf-secret -n picoctf -o yaml --insecure-skip-tls-verify
apiVersion: v1
data:
  flag: cGljb0NURntrczNjcjM3NV80MW43X3M0ZjNfM2NhZjhjMzF9Cg==
kind: Secret
metadata:
  annotations:
    kubectl.kubernetes.io/last-applied-configuration: |
      {"apiVersion":"v1","data":{"flag":"cGljb0NURntrczNjcjM3NV80MW43X3M0ZjNfM2NhZjhjMzF9Cg=="},"kind":"Secret","metadata":{"annotations":{},"name":"ctf-secret","namespace":"picoctf"},"type":"Opaque"}
  creationTimestamp: "2026-04-10T11:07:57Z"
  name: ctf-secret
  namespace: picoctf
  resourceVersion: "388"
  uid: 2295bcf8-6e15-4fa7-afa1-7dc0f455bb75
type: Opaque

Oh, we didn't get the flag, just the Base64 encoded version (we know it's Base64 cause of the text structure and the double equal sign at the end), we can use an online converter like CyberChef to decode the Base64:

recipe=From_Base64('A-Za-z0-9+/=',false,false)  # base64-to-text

- INPUT: cGljb0NURntrczNjcjM3NV80MW43X3M0ZjNfM2NhZjhjMzF9Cg==

- OUTPUT:
picoCTF{ks3cr375_41n7_s4f3_3caf8c31}

There we go! That's the flag.
