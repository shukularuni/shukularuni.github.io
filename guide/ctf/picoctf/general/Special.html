Theory

According to the description, to get the flag we have to enter into a special shell, that for some reason makes commands properly spelled and capitalized, which is odd since most commands in linux are lowercase and an abomination of letters (for example brltty-ktb and ls). So we'll have to create some creative way of looking at files.

Solution

We'll connect to the SSH with the command we are given in the challenge description:

shukularuni@SHUKULAP:~$ ssh -p 63090 ctf-player@saturn.picoctf.net
ctf-player@saturn.picoctf.net's password: 483e80d4

Special$ ls
Is
sh: 1: Is: not found

Special$ cat flag.txt
Cat flag.txt
sh: 1: Cat: not found

Okay, it seems that the program doesn't really want us seeing what's here, although there's another way of using these commands without getting that first letter capitalized, and it's using the full path of the program:

Special$ /bin/cat
Absolutely not paths like that, please!

Special$ /bin/ls
Absolutely not paths like that, please!

Hmm, very interesting, that also doesn't work. Let's try putting the command inside $() so that what's being read by the special shell is the output of our command instead of our command itself, so we can see what's actually here:

Special$ $(/bin/ls)
$(/bin/ls)
sh: 1: blargh: not found

So basically, what is happening here is when using $(ls) it first checks for files on the directory, for example if there's a flag.txt file and then executes that as a command ("flag.txt" as the command), so this way we can actually see what. So in this case there seems to be a single file here called "blargh", so let's use this same strategy to view its contents:

Special$ $(/bin/cat blargh)
$(/bin/cat blargh)
/bin/cat: blargh: Is a directory

Oh, blargh is a directory! Let's check inside that folder and see if it has the flag:

Special$ $(/bin/ls blargh)
$(/bin/ls blargh)
sh: 1: flag.txt: not found

The blargh directory has only one file, and it's the flag. Let's use that same cat command to view the flag:

Special$ $(/bin/cat blargh/flag.txt)
$(/bin/cat blargh/flag.txt)
sh: 1: picoCTF{5p311ch3ck_15_7h3_w0r57_b741d1b1}: not found

There we go! That's the flag.
