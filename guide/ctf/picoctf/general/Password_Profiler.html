Password Profiler

Name: Password Profiler
Description: We intercepted a suspicious file from a system, but instead of the password itself, it only contains its SHA-1 hash. Using OSINT techniques, you are provided with personal details about the target. Your task is to leverage this information to generate a custom password list and recover the original password by matching its hash. Download the following files: userinfo: Contains the personal details. hash: Contains the SHA-1 hash of the password. check_password: Script to test passwords against the hash.
Author: Yahaya Meddy
Tags: Easy, General Skills, picoCTF 2026
Challenge from: picoCTF 2026
Files: userinfo.txt, hash.txt, check_password.py
Hints:
1. CUPP is a Python tool for generating custom wordlists from personal data.

Theory

According to the description, to get the flag we have to generate a targeted wordlist, for this we can use CUPP (Common User Passwords Profiler), which basically creates a wordlist of any password associated to a person based on their information, in this example we have Alice and her birthdate is 1990, so common passwords might be Alice1990 or Alice1234, so CUPP creates a list of all possibilities for Alice's password, and the script we are given is supposed to check through this wordlist we have created and see which one can decrypt the hash from the hash.txt file. So we'll have to get CUPP by cloning the git.

Solution

We'll clone CUPP into our challenge folder to use the tool:

shukularuni@SHUKULAP:~/pctf26/chkpwd$ git clone https://github.com/Mebus/cupp.git

So for this program we'll need to know all of Alice's information, which we can find on the userinfo.txt file:

shukularuni@SHUKULAP:~/pctf26/chkpwd$ cat userinfo.txt
First Name: Alice
Surname: Johnson
Nickname: AJ
Birthdate: 15-07-1990
Partner's Name: Bob
Child's Name: Charlie

Now we can enter this information into CUPP to create the wordlist about Alice, and stuff we don't know that doesn't appear on the userinfo file just hit enter so that it's not filled in:

shukularuni@SHUKULAP:~/pctf26/chkpwd/cupp$ python3 cupp.py -i

Now take the alice.txt file wordlist we just created and put it besides the check_password.py program, which if we look into it we can see that it will only do a password wordlist that is named "passwords.txt", so we'll move the alice wordlist file and rename it, then run the check_password program to see if any of the passwords inside the wordlist we created works on the hash:

shukularuni@SHUKULAP:~/pctf26/chkpwd/cupp$ mv alice.txt ../passwords.txt

shukularuni@SHUKULAP:~/pctf26/chkpwd/cupp$ cd ..

shukularuni@SHUKULAP:~/pctf26/chkpwd$ python3 check_password.py
Password found: picoCTF{Aj_15901990}

There we go! That's the flag.
