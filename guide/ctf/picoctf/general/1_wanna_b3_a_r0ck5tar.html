1_wanna_b3_a_r0ck5tar

Name: 1_wanna_b3_a_r0ck5tar
Description: I wrote you another song. Put it in the picoCTF{} flag format.
Author: Alex Bushkin
Tags: Medium, General Skills, picoCTF 2019
Challenge from: picoCTF 2019
Files: lyrics.txt

Theory

According to the description, to get the flag we have to take a look at a song? This is an odd request, but alright. So we'll check out this file on the solution part to see if there might be any hidden code or code word that might be the flag.

Solution

Okay, so looking at this lyrics file I have this thought of having seen something like this before, because if I remember correctly there's an esoteric programming language called Rockstar that literally relies on the length of words to write code or hide messages, and the title of this challenge actually mentions Rockstar, so this might be it. We'll open this website I found that compiles Rockstar online:

https://codewithrockstar.com/online.html
-=- INPUT -=-
Rocknroll is right
Silence is wrong
A guitar is a six-string
...
Shout "Bring on the rock!"
Else Whisper "That ain't it, Chief"
Break it down

And if we hit the run button, in this case it's called the Rock button, we get this output:

-=- OUTPUT -=-
Unexpected 'is' at line 21 col 6: expected expression

19: They are dazzled audiences
20: Shout it!
21: Rock is electric heaven
         ^
22: Scream it!

Well that's weird. It doesn't matter though, because we can decrypt this a bit more manually, so because Rockstar is a programming language we can actually see what the code is doing:

Listen to the music - It's an input field.

If the music is a guitar - Checking that you know the value guitar.

A guitar is a six-string - a(1) six(3) string(6), guitar's value is 136.

Listen to the rhythm - Another input.

If the rhythm without Music is nothing - Music(136) minus Rhythm = 0, 136-x=0 so Rhythm is 136 too.

So that's the basic logic of this lyrics program in Rockstar, and we can do the same to get the flag, which is represented by the variables of Tommy and Music (note that They just refers to the previous variable, so that's Tommy, also Rock and Jamming count as other Music variables):

Tommy is rockin guitar - rockin(6) guitar(6) is 66.

Music is amazing sensation - amazing(7) sensation(9) is 79.

Jamming is awesome presence - awesome(7) presence(8) is 78.

Tommy is playing rock - playing(7) rock(4) is 74.

They are dazzled audiences - dazzled(7) audiences(9) is 79.

Rock is electric heaven - electric(8) heaven(6) is 86.

Tommy is jukebox god - jukebox(7) god(3) is 73.

66 79 78 74 79 86 73

As you can see it's a bunch of numbers, but this is decimal encoding. We know this because letters in decimal are in the range between 90 and 120, and we know this isn't hexadecimal cause every character has a fixed length of two numbers or letters, on this one we can see three-digit numbers like 107. So to decode this decimal we'll use an online decoding tool such as CyberChef:

recipe=From_Decimal('Space',true)  # decimal-to-text

- INPUT: 66 79 78 74 79 86 73

- OUTPUT:
BONJOVI

Perfect, that must be the code! Now we'll insert that into the picoCTF flag format to get this flag as a result:

picoCTF{BONJOVI}

There we go! That's the flag.
