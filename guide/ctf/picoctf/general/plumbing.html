plumbing

Name: plumbing
Description: Sometimes you need to handle process data outside of a file. Can you find a way to keep the output from this program and search for the flag? Connect to fickle-tempest.picoctf.net 51863.
Author: Geoffrey Njogu
Tags: Medium, General Skills, picoCTF 2019
Challenge from: picoCTF 2019
Hints:
1. Remember the flag format is picoCTF{XXXX}
2. What's a pipe? No not that kind of pipe... This kind

Theory

According to the description, to get the flag we have to search it through the output of a program inside a NetCat. We don't know much else about this program so all we can do now is check the connection, because remember this is a NetCat connection, so we'll need to add nc at the start of the server/port combination, like this:

nc fickle-tempest.picoctf.net 51863

Solution

First we'll enter into the NetCat from our command from theory:

shukularuni@SHUKULAP:~$ nc fickle-tempest.picoctf.net 51863
Again, I really don't think this is a flag
Again, I really don't think this is a flag
Again, I really don't think this is a flag
Not a flag either
Not a flag either
...
Again, I really don't think this is a flag
I don't think this is a flag either
Again, I really don't think this is a flag
I don't think this is a flag either
This is defintely not a flag

Well, that's a lot of text. But worry not! Because we can search specifically for the flag by piping this output from the NetCat over to grep, which is a command used to search for specific text, we can use this one to our advantage because we know the flag format is picoCTF{X}, so we can search that to find the flag:

shukularuni@SHUKULAP:~$ nc fickle-tempest.picoctf.net 51863 | grep picoCTF
picoCTF{digital_plumb3r_00da27CC}

There we go! That's the flag.
