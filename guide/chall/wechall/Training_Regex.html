Training: Regex

Name: Training: Regex
Tags: Training, Regex
Score: 2
Description: Your objective in this challenge is to learn the regex syntax.
Regular Expressions are a powerful tool in your way to master programming, so you should be able to solve this challenge, at least!
The solution to every task is always the shortest regular expression pattern possible.
Also note that you have to submit delimiters in the patterns too. Example pattern: /joe/i. The delimiter has to be /

Your first lesson is easy: submit the regular expression the matches an empty string, and only an empty string.

Theory

To get the solution, as the description says, we have to answer with the Regular Expression (RegEx) for nothing, ``. This is simple, all we have to do is put the starting character, which is ^, and the end character, which is $, both together. Basically just start the string and end it immediately:

^$

Solution

So let's submit this solution and see if we were right:

Answer: ^$

WeChall: Your answer is wrong.

Well, maybe it did need the solidus (/) around it. I forgot that RegEx requires using delimeters, it's these solidus characters around everything to signify that you are in fact, using RegEx. So let's put some solidus around it and submit again:

Answer: /^$/

WeChall: Correct, let`s see if you can come up with a pattern for the next problem.
Easy enough. Your next task is to submit a regular expression that matches only the string 'wechall' without quotes.

Oh, so there's more? Okay let's see. To put `wechall` in RegEx, we'll just use the same thing as before with the solidus and all that. So because ^ is the starting character and $ is the ending, then we can just put the word between these two:

Answer: /^wechall$/

WeChall: Correct, let`s see if you can come up with a pattern for the next problem.
Ok, matching static strings is not the main goal of regular expressions.
Your next task is to submit an expression that matches valid filenames for certain images.
Your pattern shall match all images with the name wechall.ext or wechall4.ext and a valid image extension.
Valid image extensions are .jpg, .gif, .tiff, .bmp and .png.
Here are some examples for valid filenames: wechall4.tiff, wechall.png, wechall4.jpg, wechall.bmp

Now we need to have both `wechall` and `wechall4` with some image extensions, first we need to put the wechall filename, with the 4 and a questionmark to make it optional. Then the extension dot with a backslash to match the literal dot. and then have the option to be any of the file extensions mentioned above:

Answer: /^wechall4?\.(jpg|gif|tiff|bmp|png)$/

WeChall: Your pattern would capture a string, but this is not wanted. Please use a non capturing group.
Your answer is wrong.

Oh right, forgot about the non-capturing groups, so we need to use `?:` inside the image formats parentheses at the start there to make it optional to choose between the file extensions instead of searching for all of them at the same time. Imagine that! So yeah, just add that:

Answer: /^wechall4?\.(?:jpg|gif|tiff|bmp|png)$/

WeChall: Correct, let`s see if you can come up with a pattern for the next problem.
It is nice that we have valid images now, but could you please capture the filename, without extension, too?
As an example: wechall4.jpg should capture/return wechall4 in your pattern now.

Oh yeah, capturing the filename without the extension, that's simple, just put a parentheses around the wechall4 filename:

Answer: /^(wechall4?)\.(?:jpg|gif|tiff|bmp|png)$/

WeChall: Your answer is correct!
Well done, this is enough for a very first lesson of regular expressions. Mission accomplished.

There we go! That's the solution.
