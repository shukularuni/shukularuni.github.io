Access

SSH: ssh oracle8@oracle.underthewire.tech -p 22

Password: multiverse111

Info

The password for oracle9 is the name of the file in the GET Request from www.guardian.galaxy.com within the log file on the desktop.

NOTE:
– Don’t include the extension.
– The password will be lowercase no matter how it appears on the screen.

Theory

To get the password, as the description says, we have to get whatever GET request was done to the website on the description from the web logs on file on the Desktop, we can use the Get-Content command to print out the text, then if it's a lot of content we'll use Select-String to search the exact webpage request.

Solution

Now that we are inside of the shell, let's see the name of the log file:

PS C:\users\Oracle8\desktop> dir

    Directory: C:\users\Oracle8\desktop


Mode                LastWriteTime         Length Name
----                -------------         ------ ----
-a----        8/30/2018   5:52 AM         405295 logs.txt

Alright, now we'll use the Get-Content with Select-String because we can see the size of the file is roughly 400KB, which would fill the terminal and would take some time to load, so we'll put the website given to us in the description on the Select-String command:

PS C:\users\Oracle7\desktop> Get-Content .\logs.txt | Select-String "www.guardian.galaxy.com"
PS C:\users\Oracle7\desktop>

Nothing huh? Maybe it's the quotes:

PS C:\users\Oracle7\desktop> Get-Content .\logs.txt | Select-String 'www.guardian.galaxy.com'
PS C:\users\Oracle7\desktop>

Nope. That's odd, let's try removing the w³ off the URL:

PS C:\users\Oracle7\desktop> Get-Content .\logs.txt | Select-String 'guardian.galaxy.com'

guardian.galaxy.com - - [28/Jul/1995:13:03:55 -0400] "GET /images/star-lord.gif HTTP/1.0" 200 786

Oh perfect, now that we have the filename requested, we just remove the extension and all other request stuff off the line, and we get the password:

star-lord
https://underthewire.tech/oracle-8
