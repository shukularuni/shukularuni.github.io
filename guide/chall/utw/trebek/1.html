Access

Info

The password for trebek2 is the name of the script referenced in a deleted task as depicted in the event logs on the desktop.

NOTE:
– Don’t include the file extension (i.e.- .vbs)
– The password will be lowercase no matter how it appears on the screen.

IMPORTANT:
Once you feel you have completed the Trebek1 challenge, start a new connection to the server, and log in with the username of Trebek2 and this password will be the answer from Trebek1. If successful, close out the Trebek1 connection and begin to solve the Trebek2 challenge. This concept is repeated over and over until you reach the end of the game.

Theory

To get the password we'll use the session we entered in the previous level, and from there we have to search through the logs of the computer to find the name of whatever file was deleted last. For that we'll have to search through the Windows Event Log file for any deleted items, to tell the log file to give us delete actions we can go to Microsoft's documentation of security auditing, where we can see that in the event of a task being deleted, it's logged with the code 4699. Then we just enter that as a filter for the Event Log file and we have this command:

Get-WinEvent -Path .\EventLogsFilename.evtx -FilterXPath "*[System[EventID=4699]]"

Solution

Because we're already inside of the PowerShell from the previous level, let's see the directory to know the name of the Event Log file:

Now we can just select the last one of these to find our desired event, with the Select-Object command, and put the Message preperty at the end to view the whole message:

PS C:\users\trebek1\desktop> (Get-WinEvent -Path .\security.evtx -FilterXPath "*[System[EventID=4699]]" | Select-Object -Last 1).Message

And we grab the command that was executed in there:

powershell.exe -NonInteractive -NoLogo -NoProfile -File 'c:\users\trebek1\mess_cleaner.ps1'

Just remove everything but the filename (file extension included), and we get the password:

mess_cleaner
