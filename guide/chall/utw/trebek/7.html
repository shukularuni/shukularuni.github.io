Access

SSH: ssh trebek7@trebek.underthewire.tech -p 22

Password: 40_reader

Info

The password for trebek8 is the name of the program set to run prior to login if sticky keys are activated PLUS the name of the file on the desktop.

NOTE:
– Don’t include the file extension (i.e.- .vbs). If the name is “script.vbs” and the file on the desktop is named “1234”, then the password would be “script1234”.
– The password will be lowercase no matter how it appears on the screen.

▼ HINT:

The registry sounds like a good place to look!

▼ HINT:

I heard something about “Image File Execution Options”…

Theory

To get the password, as the description says, we have to look through the registry to find what program runs right before login if sticky keys is activated, so if that's the case, we can find the program in the debugger section of whatever it depends on. First we'll need to know if Sticky Keys is activated, if so then we'll look through the program that controls Sticky Keys to track the debugger, which is the program being run right before log in, the program that runs Sticky Keys is also known as sethc.exe, and this program can be found inside the "Image File Execution Options" folder of the local machine's registry keys, which aligns with the second hint of the description.

Solution

Now that we are inside of the shell, we can use dir to view the name of the file in the desktop:

PS C:\users\trebek7\desktop> dir

    Directory: C:\users\trebek7\desktop


Mode                LastWriteTime         Length Name
----                -------------         ------ ----
-a----        8/30/2018  10:47 AM              0 99

So now we'll check that Sticky Keys is up and running by checking through the registry on the Accessibility folder:

PS C:\users\trebek7\desktop> Get-Item 'HKCU:\Control Panel\Accessibility\StickyKeys'

    Hive: HKEY_CURRENT_USER\Control Panel\Accessibility


Name                           Property
----                           --------
StickyKeys                     Flags : 510

That 510 number just means that the program is indeed running. Now we just have to check the sethc.exe program to know what's running right before the log in through the debugger property. So we'll look for the program in the folder from theory that we confirmed with the second hint of the description, and look at all its properties:

PS C:\users\trebek7\desktop> Get-Item 'HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\sethc.exe' | Get-ItemProperty

Debugger     : han_solo.exe
PSPath       : Microsoft.PowerShell.Core\Registry::HKEY_LOCAL_MACHINE\SOFTWA... 
PSParentPath : Microsoft.PowerShell.Core\Registry::HKEY_LOCAL_MACHINE\SOFTWA... 
PSChildName  : sethc.exe
PSDrive      : HKLM
PSProvider   : Microsoft.PowerShell.Core\Registry

Now just grab the debugger filename, remove the .exe app extension, and add the name of the file in the desktop:

han_solo99
