Access

SSH: ssh trebek4@trebek.underthewire.tech -p 22

Password: 10.30.1.18address

Info

The password for trebek5 is the last execution date of Microsoft Access PLUS the name of the text file on the user’s desktop.

Note:
– Format for the date is 2 digit month, 2 digit day, 4 digit year. Ex: 9 feb 2009 would be 02/09/2009.
– If the date is “02/09/2009” and the file on the desktop is named “_bob”, the password would be “02/09/2009_bob”.

https://technet.microsoft.com/en-us/library/ee176852.aspx

▼ HINT:

https://www.fireeye.com/blog/threat-research/2013/08/execute.html

Theory

To get the password, as the description says, we have to get the date of the last time Mircosoft Access was executed. The problem is that there's not really any command I know of that can look at registries of uninstalled applications, although there is a file in Windows that tracks this, and it's Prefetch (inside the Windows folder). And we can use the path of the file to look for anything that mentions the word "Access".

Solution

Now that we are inside of the shell, we can use dir to view the name of the file in the desktop:

PS C:\users\trebek4\desktop> dir

    Directory: C:\users\trebek4\desktop


Mode                LastWriteTime         Length Name
----                -------------         ------ ----
-a----        8/30/2018  10:46 AM              0 _red

So now we can use the Get-ChildItem command to search through the Prefetch file for any mention of the word "Access":

PS C:\users\trebek3\desktop> Get-ChildItem "C:\Windows\Prefetch" -Filter '*Access*"

    Directory: C:\Windows\Prefetch


Mode                LastWriteTime         Length Name
----                -------------         ------ ----
-a----         1/5/2017   6:04 PM          65058 MSACCESS.EXE-EF45328A.pf

Since there's only one execution of Microsoft Access, then that's the last time it was executed, now just get the date from there and add some zeroes in for the correct format of the date and add in the name of the file in the desktop to get the password:

01/05/2017_red
