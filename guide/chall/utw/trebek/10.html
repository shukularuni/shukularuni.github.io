Info

The password for trebek11 is the last name of the user who enabled Oni-Wan Kenobi’s account as depicted in the event logs on the desktop PLUS the name of the file on the desktop.

NOTE:
– If the last name of the user is “johnson” and the file on the desktop is named “1234”, the password would be “johnson1234”.
– The password will be lowercase no matter how it appears on the screen.

▼ HINT:

https://msdn.microsoft.com/en-us/powershell/reference/5.1/microsoft.powershell.diagnostics/get-winevent

Theory

To get the password, as the description says, we have to look through a Windows Event Log file again, but now search for whoever enabled Kenobi's Windows account. So, to search for that log we have to look for the event code of 4722, which is the code to an account being enabled. This command will look like all the others we have done with Event Logs in previous levels to this one, because we don't know how specific we have to get, we'll create two commands, the first one is the normal search for the event code, and if there's too many of those, then the second one will search for specifically logs that contain the word "kenobi" in them to search specifically for the enabling of Kenobi's account:

Get-WinEvent -Path .\FileNameOfTheFile.evtx -FilterXPath "*[System[EventID=4722]]"
(Get-WinEvent -Path .\FileNameOfTheFile.evtx -FilterXPath "*[System[EventID=4722]]").Where({$_.Message -like "*kenobi*"})

Solution

Now that we are inside of the shell, we can use dir to view the name of the file in the desktop and the name of the Event Log file:

PS C:\users\trebek10\desktop> dir

    Directory: C:\users\trebek10\desktop


Mode                LastWriteTime         Length Name
----                -------------         ------ ----
-a----        8/30/2018  10:48 AM              0 2121
-a----        8/30/2018   5:54 AM       99684352 security.evtx

And if we put the name of the file into our first command in theory; yup, we just get too many of these logs:

PS C:\users\trebek10\desktop> Get-WinEvent -Path .\security.evtx -FilterXPath "*[System[EventID=4722]]"

   ProviderName: Microsoft-Windows-Security-Auditing

TimeCreated                     Id LevelDisplayName Message
-----------                     -- ---------------- -------
5/11/2017 8:36:52 PM          4722 Information      A user account was enabl...
5/11/2017 6:56:12 PM          4722 Information      A user account was enabl...
5/11/2017 6:46:25 PM          4722 Information      A user account was enabl...
5/11/2017 6:26:08 PM          4722 Information      A user account was enabl...

So now let's go with the second command of theory to see if we can reduce a bit our options:

PS C:\users\trebek10\desktop> (Get-WinEvent -Path .\security.evtx -FilterXPath "*[System[EventID=4722]]").Where({$_.Message -like "*kenobi*"})

   ProviderName: Microsoft-Windows-Security-Auditing

TimeCreated                     Id LevelDisplayName Message
-----------                     -- ---------------- -------
5/11/2017 6:56:12 PM          4722 Information      A user account was enabl...

Just one log left! Perfect, now we can just add the Message property at the end to view the full message:

PS C:\users\trebek10\desktop> (Get-WinEvent -Path .\security.evtx -FilterXPath "*[System[EventID=4722]]").Where({$_.Message -like "*kenobi*"}).Message
A user account was enabled.

Subject:
        Security ID:            S-1-5-21-3968311752-1263969649-2303472966-1153
        Account Name:           admiral.ackbar
        Account Domain:         UNDERTHEWIRE
        Logon ID:               0x12CEFCA

Target Account:
        Security ID:            S-1-5-21-3968311752-1263969649-2303472966-1154
        Account Name:           obi-wan.kenobi
        Account Domain:         UNDERTHEWIRE

So now just take last name of the account name that enabled Kenobi's account and add the name of the file in the desktop to get the password:

ackbar2121
