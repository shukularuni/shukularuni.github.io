Theory

It's Base64 again, but we know from the previous challenge that it's because they're multiple layers of encryption. So, looking at the structure of the string, we can see a couple A and solidus characters, which means this has binary characters when decoding, that might be interesting to look into. So let's decode it.

Solution

Let's put it into CyberChef to decode from Base64 to check what it really is:

recipe=From_Base64('A-Za-z0-9+/=',true,false)  # base64-to-text

- INPUT: cEJBVEVOR0hZTkdWQkFGDABMQkgAVU5JUgBQQlpDWVJHUlEAcExPUkVwVVJTAFBVTllZUkFUUgADFwEqKmdVVkYAUFVO...

- OUTPUT:
pBATENGHYNGVBAF..LBH.UNIR.PBZCYRGRQ.pLORE...

Alright that seems to be text that's been rotated with a shift of 13 in the caesar cipher just like the previous challenge, but it has weird binary characters everywhere, so let's also put URL encode so that we can see what the function of these characters are (I'm using URL encode so that it only encodes special characters like the binary thingies):

recipe=From_Base64('A-Za-z0-9+/=',true,false)ROT13(true,true,false,13)Swap_case()URL_Encode(false)  # base64-to-text, rotation 13, swap case, and url-percent-encode

- INPUT: cEJBVEVOR0hZTkdWQkFGDABMQkgAVU5JUgBQQlpDWVJHUlEAcExPUkVwVVJTAFBVTllZUkFUUgADFwEqKmdVVkYAUFVO...

- OUTPUT:
cONGRATULATIONS%0C%00YOU%00HAVE%00COMPLET...

Let's also swap case becuase the first letter is lowercase and the others are uppercase, and it seems the null character (%00) seems to be space, while the new line is an asterisk, so let's replace those:

recipe=From_Base64('A-Za-z0-9+/=',true,false)Gunzip()ROT13(true,true,false,13)Swap_case()URL_Encode(false)Find_/_Replace({'option':'Simple string','string':'*'},'\\n',true,false,true,false)Find_/_Replace({'option':'Simple string','string':'%00'},' ',true,false,true,false)  # base64-to-text, rotation 13, swap case, url-percent-encode, replace * for newline, and replace %00 for [space]

- INPUT: cEJBVEVOR0hZTkdWQkFGDABMQkgAVU5JUgBQQlpDWVJHUlEAcExPUkVwVVJTAFBVTllZUkFUUgADFwEqKmdVVkYAUFVO...

- OUTPUT:

Now we're almost there, I can see that the %17 is going to be 7 because of the challenge number we already know, so it's safe to say that any character in the 10-19 range is their equivalent number in the 0-9 range. So we can replace all the numbers:
